Для создания локально-удаленного репозитория, нужно при инициализации (создании) репозитория указать флаги "--bare" (значит репозиторий локально-удаленный), "--shared=groop" (значит что разрешаются внесения изменений с склонированного репозитория (например команды "git push") на этот локально-удаленный репозиторий). Так же при инициализации локально-удаленного репозитория в конце имени этого репозитория обязательно указывается ".git" (например "test-repo.git"). Пример:

	git init --bare --shared=group test-repo.git

Для клонирования этого репозитория, нужно ввести команду:

	git clone 'C:\Users\ADMIN\Desktop\MyRepos\test-repo.git'

	При клонировании не обязательно в конце адреса директории ставить ".git".

	Дальше для работы над склонированным репозиторием, нужно на него переключиться:

		cd test-repo

Для того, чтобы выполнялись комманды "git push --force" от лица клонированных репозиториев, нужно настроить файл конфигурации "config", которай находится непосредственно в самом локально-удаленном репозитории. В этом файле нужно изменить значение свойства "denyNonFastforwards" с "true" на "false".


Для того, чтобы выполнять команды 'git' в локально-удаленном репозитории, нужно переключиться в консоли на эту директорию (на сам репозиторий) (обязательно с окончанием ".git"). Например:

	cd test-repo.git
	git log

Будучи в директории локально-удаленного репозитория к сожалению нельзя добавлять/измменять/просматриваит его файлы. Для этого нужно клонировать этот репозиторий.
